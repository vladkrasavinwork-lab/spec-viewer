# Review self-check

Before finalization, verify:

- Read the full specification and searched related sections before asserting absence/conflict.
- Every issue describes one material problem and is not a duplicate or style preference.
- Every issue includes the nearest section/heading and a short excerpt, or explicitly null evidence
  when absence across the document is the basis.
- Severity reflects impact; confidence reflects evidence quality.
- Every `decision_required` issue that blocks progress references an open question.
- All referenced question IDs exist; answered questions are not repeated.
- Summary counts exactly match issue and clarification registers.
- Readiness follows the rubric and the narrative agrees with structured artifacts.
- Conversion uncertainty is separated from source-document quality.
- No source, normalized document, or completed run was edited.
