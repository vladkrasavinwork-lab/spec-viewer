# Rewrite self-check

- Read all source and latest review artifacts; verify the review input hashes still match.
- Preserve every confirmed source requirement or record its modification/removal.
- Ensure each applied change has one clear rationale and valid provenance.
- Ensure every confirmed decision references an answered question ID.
- Keep every unanswered decision in `unresolved-questions.md`; do not imply a chosen option.
- Ensure assumption wording remains conditional and all unconfirmed assumptions are counted.
- Ensure every changed/new/removed requirement has a traceability entry.
- Check requirement IDs for uniqueness, stability, and domain consistency.
- Check revised text for hidden scope expansion, invented facts, and contradictions.
- Validate all YAML and finalize only through the deterministic CLI.
